# Язык программирования «Гойда»

Русскоязычный учебный язык программирования с C-подобными блоками, функциями, классами, модулями, исключениями, потоками и базовой интеграцией с native-библиотеками.

## Сборка и запуск

[![Linux CI](https://github.com/Artem468/goida_lang/actions/workflows/linux_build.yml/badge.svg)](https://github.com/Artem468/goida_lang/actions/workflows/linux_build.yml)
[![Windows CI](https://github.com/Artem468/goida_lang/actions/workflows/windows_build.yml/badge.svg)](https://github.com/Artem468/goida_lang/actions/workflows/windows_build.yml)

```bash
git clone <url>
cd goida_lang

cargo build --release -p cli -p lsp
cargo run -p cli -- run examples/simple.goida
cargo run -p cli -- repl
cargo test
```

## Базовый синтаксис

```goida
имя = "Иван"
возраст: число = 25
активен: логический = истина
ничего = пустота

если (возраст >= 18 и активен) {
    печать("доступ разрешен")
} иначе {
    печать("доступ закрыт")
}
```

Основные типы: `число`, `дробь`, `строка`, `логический`, `пустота`, `список`, `массив`, `словарь`, `указатель`, `неизвестно`. Для встроенных классов доступны имена `Строка`, `Список`, `Массив`, `Словарь`, `Файл`, `Терминал`, `Система`, `ДатаВремя`, `Поток`, `Мьютекс`, `БлокировкаЧтенияЗаписи`.

Логические операции пишутся словами: `и`, `или`. Остальные операторы привычные: `+`, `-`, `*`, `/`, `%`, `==`, `!=`, `>`, `<`, `>=`, `<=`, `!`, `=`.

Составные присваивания можно использовать как обычные инструкции для переменных, полей и индексов: `+=`, `-=`, `*=`, `/=`, `%=`.

```goida
число = 10
число += 5
число %= 4

объект.поле *= 2
список[0] -= 1
```

## Функции

Параметры могут иметь тип и значение по умолчанию. При вызове поддерживаются позиционные и именованные аргументы; именованные идут после позиционных.

```goida
функция суммировать(a: число, b: число = 2) -> число {
    вернуть a + b
}

печать(суммировать(3))          // 5
печать(суммировать(3, 4))       // 7
печать(суммировать(b=10, a=1))  // 11
```

## Циклы

```goida
счетчик = 0
пока (счетчик < 3) {
    печать(счетчик)
    счетчик = счетчик + 1
}

для (i = 1, i < 4, i += 1) {
    печать(i)
}
```

## Классы

Классы поддерживают наследование, публичные и приватные поля/методы, статичные поля/методы, конструкторы и создание объектов через `новый`.

```goida
класс Человек {
    приватный имя: строка = ""
    публичный статичный количество: число = 0

    публичный конструктор создать(имя: строка) {
        это.имя = имя
        Человек.количество = Человек.количество + 1
    }

    публичный функция получить_имя() -> строка {
        вернуть это.имя
    }
}

человек = новый Человек("Иван")
печать(человек.получить_имя())
печать(Человек.количество)
```

Наследование указывается после имени класса:

```goida
класс ОшибкаПроверки(Ошибка) {
}
```

## Модули и импорт

Файл подключается строковым путем и получает локальный псевдоним.

```goida
подключить "math_module.goida" в математика

печать(математика.сложить(2, 3))
объект = новый математика.Калькулятор()
```

## Исключения

Для обработки ошибок используются `попробовать`, один или несколько блоков `перехватить` и оператор `выбросить`.

```goida
класс ОшибкаПроверки(Ошибка) {
}

функция проверить(значение: число) {
    если (значение < 0) {
        выбросить ОшибкаПроверки("значение меньше нуля")
    }
}

попробовать {
    проверить(-3)
} перехватить (ОшибкаПроверки как сообщение) {
    печать("перехвачено: " + сообщение)
} перехватить (Ошибка как сообщение) {
    печать("общая ошибка: " + сообщение)
}
```

Форма `перехватить (как сообщение)` ловит любую ошибку и сохраняет текст в переменную. Форма `перехватить { ... }` ловит любую ошибку без привязки сообщения.

## Потоки

Блок `поток` выполняет код в фоне. Для синхронизации доступны классы `Поток`, `Мьютекс`, `БлокировкаЧтенияЗаписи`, а также системные функции вроде `Система.сон(...)`.

```goida
значение = 1

поток {
    значение = 2
}

Система.сон(50)
печать(значение)
```

## Native FFI

Native-библиотеки описываются декларацией `библиотека`. Внутри перечисляются функции и глобальные переменные, экспортируемые динамической библиотекой.

```goida
библиотека "target/debug/native_ffi_test_lib.dll" {
    функция add(a: число, b: число) -> число {}
    переменная counter: число
}

печать(add(2, 3))
```

На Linux и macOS путь должен указывать на соответствующий `.so` или `.dylib`.

## Коллекции и JSON

Встроенные классы `Список`, `Массив` и `Словарь` дают изменяемые списки, неизменяемые массивы и словари. JSON-функции доступны через встроенный модуль/класс JSON, примеры находятся в `examples/json_test.goida` и `examples/json_roundtrip_test.goida`.

```goida
список = новый Список()
список.добавить(10)
печать(список[0])
```

## Ключевые слова

| Слово | Назначение |
| --- | --- |
| `подключить ... в ...` | импорт файла как модуля |
| `функция` | объявление функции или метода |
| `вернуть` | возврат значения |
| `класс` | объявление класса |
| `новый` | создание объекта |
| `конструктор` | конструктор класса |
| `это` | текущий объект внутри метода |
| `публичный`, `приватный` | модификаторы доступа |
| `статичный` | статичный член класса |
| `если`, `иначе` | ветвление |
| `пока`, `для` | циклы |
| `попробовать`, `перехватить`, `выбросить` | исключения |
| `поток` | фоновое выполнение блока |
| `библиотека`, `переменная` | декларации native FFI |
| `истина`, `ложь`, `пустота` | литералы |
